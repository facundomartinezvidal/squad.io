## Architecture

**Stack:** Next.js 15 (App Router) + Convex + Better Auth + TypeScript + Tailwind CSS 4

### Convex (Backend)

All backend code lives in `convex/`. Convex auto-deploys on file save when `npx convex dev` is running. No manual migrations тАФ schema changes apply automatically.

- `convex/schema.ts` тАФ App data tables (currently empty, auth tables managed by component)
- `convex/auth.ts` тАФ Better Auth config with email+password, exports `authComponent` and `createAuth`
- `convex/http.ts` тАФ HTTP router registering auth routes
- `convex/auth.config.ts` тАФ Convex auth provider (JWT via better-auth)
- `convex/convex.config.ts` тАФ Registers `@convex-dev/better-auth` component

Auth tables (user, session, account, verification, jwks, rateLimit) are managed internally by the `@convex-dev/better-auth` component тАФ do NOT define them in `convex/schema.ts`.

## Product

Squad.io connects players who have an incomplete group with available players looking to join a match. Full product definition at `.claude-dev-flow/product-definition.md`.

**Core matchmaking criteria:** Rank > Role/Position > Language > Server/Region

**Key features:** Quick search with filters, player profiles (game API stats), real-time chat, reputation system, match history, favorites list.

**Governance:** Zero tolerance for toxicity, match ghosting, and smurfing/rank falsification.

## Code Style

- Path alias: `~/` maps to `src/` (e.g., `import { cn } from "~/lib/utils"`)
- `verbatimModuleSyntax: true` тАФ Use `import type` for type-only imports
- ESLint enforces consistent type imports and unused vars with `_` prefix allowed
- shadcn components use `new-york` style variant
