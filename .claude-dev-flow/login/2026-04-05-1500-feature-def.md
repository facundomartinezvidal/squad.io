# Feature: Login Funcional
Date: 2026-04-05
Status: Approved
Feature Branch: `feat/login`
Base Branch: main

## Business Context
**Problem Statement**: La infraestructura de auth está configurada en Convex pero el frontend no tiene conexión. Los usuarios no pueden autenticarse.
**Target Users**: Jugadores que quieren usar Squad.io para encontrar compañeros de equipo.
**Success Metrics**: Login end-to-end funcional con email+password, middleware protegiendo rutas, logout funcional.
**Priority**: P0 — sin auth no se puede avanzar con ninguna feature del producto.

## Scope
- Solo login (email + password)
- Sin signup page (seed script para usuario de prueba)
- Sin forgot/reset password
- Middleware activo protegiendo rutas
- Post-login redirect a `/`
- Landing page muestra estado autenticado + botón logout

## Functional Requirements

### Happy Path
1. Usuario no autenticado visita `/` → middleware redirige a `/login`
2. Usuario ve formulario de login (email + password)
3. Usuario ingresa credenciales válidas → submit
4. `authClient.signIn.email()` se ejecuta contra `/api/auth/*` → proxy a Convex
5. Convex valida credenciales → retorna session token
6. Cliente almacena session cookie
7. Redirect a `/` → landing page muestra "Find your squad" + botón "Sign out"

### Error Paths
1. Credenciales inválidas → mostrar "Invalid email or password" en el formulario
2. Email vacío → validación client-side "Email is required"
3. Email formato inválido → validación client-side "Invalid email format"
4. Password vacío → validación client-side "Password is required"
5. Password < 8 chars → validación client-side "Password must be at least 8 characters"
6. Network error → mostrar error genérico
7. Usuario autenticado visita `/login` → middleware redirige a `/`

### Logout
1. Usuario click "Sign out" en landing page
2. `authClient.signOut()` se ejecuta
3. Session cookie se elimina
4. Redirect a `/login`

## Non-Functional Requirements

### Performance
- Login form render: < 500ms
- Auth request: < 2s (dependiente de Convex)
- Middleware check: < 50ms (cookie check local, sin network)

### Security
- Credenciales nunca en URL, siempre POST body
- Session cookie httpOnly (manejado por Better Auth)
- CSRF protection (manejado por Better Auth)
- No almacenar credenciales en client state después del submit

## Technical Implementation

### Tech Stack
- **Form**: TanStack React Form (`@tanstack/react-form`)
- **UI**: shadcn/ui (Button, Card, Input, Label)
- **Auth Client**: Better Auth React client con plugin `convexClient()`
- **Auth Server**: `@convex-dev/better-auth/nextjs` helpers
- **Provider**: `ConvexBetterAuthProvider`

## Edge Cases
1. Double-click submit button → disabled while submitting (isSubmitting state)
2. Session expires while on protected page → next navigation triggers middleware redirect
3. Browser back button after login → should stay on `/` (middleware handles)
4. Browser back button after logout → should redirect to `/login` (middleware handles)
5. Direct URL access to `/login` while authenticated → redirect to `/`
6. Convex dev server not running → auth requests fail → show error message
