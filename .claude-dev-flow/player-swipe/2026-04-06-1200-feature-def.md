# Feature: Player Card Swipe (LoL Matchmaking MVP)
Date: 2026-04-06
Status: Approved
Feature Branch: `feat/player-swipe`
Base Branch: main

## Business Context
**Problem Statement**: Squad.io has no core feature yet. Users who log in see a placeholder page. The matchmaking experience — the app's core transaction — doesn't exist.
**Target Users**: Competitive LoL players looking for teammates to fill their squad right now.
**Success Metrics**: Users can browse available players, like/reject them, and mutual matches are detected.
**Priority**: P0 — this is the first real feature of the product.

## Functional Requirements

### User Stories
1. As an authenticated player, I want to see available LoL players one at a time so I can decide if I want to play with them.
   - Acceptance: A player card appears showing username, rank, role, level, winrate, language, server, reputation.
2. As a player, I want to like or reject another player so my preferences are recorded.
   - Acceptance: Clicking Like/Reject saves a swipe to the DB and shows the next player.
3. As a player, I want to know when there's a mutual like so I can connect.
   - Acceptance: When both players have liked each other, a "It's a match!" indicator appears.

### Detailed Behavior

#### Happy Path
1. User logs in and lands on the home page
2. System queries for the next unswiped online player
3. Player card renders with all LoL-specific data
4. User clicks Reject (X) → swipe saved as "reject", next card loads
5. User clicks Like (heart) → swipe saved as "like"
   - If target had already liked current user → "It's a match!" overlay
   - Otherwise → next card loads
6. When no more players are available → empty state message

#### Error Paths
1. User not authenticated → "Sign in" message with link to /login
2. No player profile exists for user → auto-create minimal profile on first visit
3. No online players available → "No more players available" empty state
4. Network error on swipe → button stays enabled, user can retry

### State Model
```
[Loading] → [Not Authenticated] → redirect to /login
[Loading] → [No Profile] → auto-create → [Swiping]
[Swiping] → Like → [Match Detected] → dismiss → [Swiping]
[Swiping] → Like → [Swiping] (no match)
[Swiping] → Reject → [Swiping]
[Swiping] → no more players → [Empty State]
```

## Data Model

### `players` table
| Field | Type | Notes |
|-------|------|-------|
| username | string | Display name |
| avatarUrl | optional string | Profile image URL |
| rank | string | Iron/Bronze/Silver/Gold/Platinum/Emerald/Diamond/Master/Grandmaster/Challenger |
| division | optional string | I/II/III/IV (only Iron through Diamond) |
| role | string | Top/Jungle/Mid/ADC/Support |
| level | number | Summoner level (1-500+) |
| winRate | number | Percentage 0-100 |
| language | string | es/en/pt etc |
| server | string | LAS/LAN/NA/EUW/EUNE/KR etc |
| reputation | number | 0-5 rating |
| isOnline | boolean | Simulated for now |
| isLookingForMatch | boolean | Filter flag |
| tokenIdentifier | string | Links to auth identity via `ctx.auth.getUserIdentity().tokenIdentifier` |

**Indexes:**
- `by_tokenIdentifier` on `[tokenIdentifier]` — look up player by auth identity
- `by_online_status` on `[isOnline, isLookingForMatch]` — query available players

### `swipes` table
| Field | Type | Notes |
|-------|------|-------|
| swiperId | Id<"players"> | Who swiped |
| targetId | Id<"players"> | Who was swiped on |
| direction | "like" \| "reject" | Swipe direction |

**Indexes:**
- `by_swiper` on `[swiperId]` — get all swipes by a player
- `by_swiper_target` on `[swiperId, targetId]` — check if already swiped (dedup + match check)
- `by_target_direction` on `[targetId, direction]` — check who liked a player

No explicit timestamp — Convex auto-provides `_creationTime`.

## Backend Functions

### `convex/players.ts`
- **`getCurrentPlayer`** (query): Get current user's player doc via `tokenIdentifier`. Returns player doc or `null`.
- **`getNextPlayer`** (query): Reactive query. Gets all swipes by current player, builds Set of swiped IDs, iterates online+looking players, returns first unswiped non-self player. Returns `null` when exhausted.
- **`createMyProfile`** (mutation): Creates player record for current authenticated user with default values. Called on first visit if no profile exists.

### `convex/swipes.ts`
- **`recordSwipe`** (mutation): Args: `targetId: Id<"players">`, `direction: "like" | "reject"`. Idempotent (checks existing swipe first). If "like", checks for reciprocal like via `by_swiper_target` index. Returns `{ matched: boolean }`.
- **`getMatches`** (query): Returns all mutual matches for current player. Fetches own "like" swipes, checks each for reciprocal. Returns matched player docs. Bounded with `.take(50)`.

### `convex/seed.ts` (extend existing)
- **`seedPlayers`** (internalMutation): Creates ~10 fake LoL player profiles with varied:
  - Ranks: Silver through Diamond
  - Roles: all 5 represented
  - Servers: LAS, LAN, NA, EUW mix
  - Languages: es, en, pt mix
  - Win rates: 45-68%
  - Levels: 30-350
  - Reputation: 3.0-5.0
  - All `isOnline: true`, `isLookingForMatch: true`
  - Fake `tokenIdentifier` like `"seed|player-1"` (won't collide with real auth)

## Non-Functional Requirements

### Performance
- Card loads instantly (Convex reactive query)
- Swipe mutation < 100ms
- No pagination needed for MVP (~10 seed players)

### Security
- All queries/mutations verify auth via `ctx.auth.getUserIdentity()`
- Player identity derived server-side, never from client args
- Swipe deduplication prevents spam

### Reliability
- Convex mutations are transactional — no double-swipe race conditions
- Reactive queries auto-update when data changes

## Edge Cases
1. **Double-tap on Like/Reject**: Mutation is idempotent — checks existing swipe before inserting
2. **User with no player profile**: Auto-create on first visit via `createMyProfile` mutation
3. **No online players**: Empty state shown
4. **Self-swipe**: `getNextPlayer` skips current user
5. **Already swiped all players**: Empty state when no more unswiped players
6. **Auth token expired mid-swipe**: Convex auth check fails, error shown
7. **Concurrent likes from both sides**: Convex transactional mutations handle this — second mutation detects the match

## Out of Scope (Explicitly)
- Real-time notifications when matched
- Profile edit UI
- Real online tracking (heartbeat/presence)
- Game API integration (Riot API)
- Chat/messaging between matched players
- Swipe animations/gestures (touch swipe)
- Filtering by rank/role/server before swiping
- Signup flow (only login exists)
