# Feature: Migrate to Better Auth + Convex (Full Stack Replacement)
Date: 2026-04-05
Status: Approved
Feature Branch: `feat/better-auth-convex-migration`
Base Branch: `main`

## Business Context
**Problem Statement**: Replace the entire backend stack (Supabase + Drizzle/PostgreSQL + tRPC) with Convex + better-auth. Remove all demo data (posts). The template becomes a clean Next.js + Convex + better-auth starter.
**Target Users**: Developers using this as a starter template.
**Success Metrics**: Auth flow works end-to-end. Opening the app redirects to /login if not authenticated. No remnants of old stack. Clean template ready for new features.
**Functional on delivery**: YES - the auth must be fully working, not just scaffolded. Opening the app must redirect to /login. Signup/login must work. The landing page (/) is a protected route.
**Priority**: P1

## Architecture Overview

**Before:**
Next.js -> tRPC -> Drizzle -> PostgreSQL + Supabase Auth (disabled)

**After:**
Next.js -> Convex (queries/mutations) + better-auth (inside Convex HTTP actions)

`@convex-dev/better-auth` runs better-auth **inside Convex HTTP actions**:
- Auth API lives at `*.convex.site`, proxied through Next.js `/api/auth/[...all]`
- Auth tables (user, session, account, verification) stored in Convex
- Session verification from Next.js uses cookie-based checks
- Client uses Convex reactive hooks (`useQuery`, `useMutation`) instead of tRPC

## Edge Cases and Error Scenarios

### Data Edge Cases
1. **Duplicate email signup**: better-auth returns error, UI shows message
2. **Empty form fields**: Client-side validation prevents submission
3. **Very long inputs**: better-auth has built-in limits
4. **Unicode in name**: Convex stores UTF-8 natively

### Timing Edge Cases
5. **Double-click submit**: Disable button during submission
6. **Session expires mid-use**: Next request triggers middleware redirect to /login
7. **Simultaneous login from two tabs**: Both get valid sessions
8. **Convex cold start**: First auth request may take ~1-2s extra

### Network Edge Cases
9. **Convex down during login**: Proxy returns error, UI shows message
10. **Missing BETTER_AUTH_SECRET**: Convex function fails with clear error
11. **Missing NEXT_PUBLIC_CONVEX_URL**: Build-time error

### User Behavior Edge Cases
12. **Navigate to /login while logged in**: Middleware redirects to /
13. **Navigate to protected route while logged out**: Middleware redirects to /login
14. **Browser cookies disabled**: Auth won't work
15. **Back button after logout**: Middleware catches, redirects to /login
16. **Forgot password without email service**: Logs to console (dev mode)

### Developer Experience Edge Cases
17. **Only one dev server running**: Clear error about missing Convex connection
18. **`npx convex dev` not initialized**: Build fails with helpful error
19. **Old .env vars still present**: No conflict (unused vars ignored)
20. **Existing PostgreSQL data**: Irrelevant (no migration needed, data deleted)

## Out of Scope

3. Role-based access control
4. Email sending service (forgot password logs to console)
5. User profile page

7. Rate limiting customization

## Developer Setup Notes (for template users)
1. `npm install`
2. `npx convex dev` (creates Convex project, follow prompts - keep running)
3. `npx convex env set BETTER_AUTH_SECRET $(openssl rand -base64 32)`
4. `npx convex env set SITE_URL http://localhost:3000`
5. Add to `.env.local`: `NEXT_PUBLIC_CONVEX_SITE_URL=https://<deployment>.convex.site`
6. In separate terminal: `npm run dev`
